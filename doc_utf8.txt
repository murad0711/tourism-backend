User Story: Digital Tourism Permission System
User Persona:
Rahim Uddin, a young professional from Dhaka, plans to visit Sajek Valley with his friends. Since Sajek is a restricted tourism area, visitors must obtain official travel permission before entering. Instead of going through the traditional manual process, Rahim uses the Digital Tourism Permission System to apply online.
He creates an account, fills out the tourism permit application form, uploads his National ID, and pays the permit fee through a digital payment gateway. The system records his application and sends it to the tourism authority for review.
At the administration office, Captain Farhana Ahmed, a tourism security officer, reviews Rahim’s application through the admin dashboard. She verifies the submitted information and documents. If everything is correct, she approves the request; otherwise, she can request corrections or reject the application.
Once approved, the system generates a digital permit with a QR code, which Rahim can download and show at security checkpoints during his trip.At a higher level, Dr. Mahmud Hasan, a representative of the national tourism authority, monitors tourism activities through the system’s analytics dashboard. He can track visitor statistics, tourism demand, and revenue reports to support policy decisions and improve tourism management.
Through this system(web and mobile app), tourists can easily obtain travel permits, authorities can manage applications efficiently, and the government can monitor tourism activity in a structured and transparent way.
User Journey:
User Registration
User Action :
Rahim visits the Digital Tourism Permission System website and creates an account by providing:
* Full Name
* Mobile Number
* Email Address
* Password
* Tourist Type (Domestic / Foreign)
He verifies his account using an OTP sent to his phone or email. He enters the OTP to verify his account.


System Response: The system automatically:
* validates the registration information
* verifies the OTP
* creates a new user account
* assigns a unique visitor ID
* stores user details in the database
* redirects the user to the tourist dashboard








Profile Update (Tourist)
User Action (Tourist):
Rahim logs into the Digital Tourism Permission System and opens his Tourist Dashboard.
From the dashboard, he navigates to the My Profile section to review his personal information.
Rahim notices that he needs to update his contact information. He selects Edit Profile and updates the following details:
• Mobile Number
• Email Address
After entering the new information, he clicks Save Changes.




System Response: 
* Validates the updated information
* Checks whether the mobile number and email format are valid
* Sends an OTP verification code to the new mobile number or email
Rahim enters the OTP to verify the updated contact information.
Once verification is successful, the system:
* Updates the user profile in the database
* Confirms that the profile has been successfully updated
* Displays a confirmation message:
“Your profile information has been successfully updated.”

Application Review

An army officer or tourism authority officer logs into the Admin Dashboard. The admin opens the Pending Applications section and reviews Rahim's application.

Admin has the following options:

Resubmission Scenario

The admin finds that the NID copy is unclear. Admin selects Request Resubmission and adds remarks: "Please upload a clearer NID copy."


System Response:
            * updates application status to Resubmission Requested
            * sends notifications via:
            * In-app alert

The visitor receives the notification.


User Action (Visitor)
Rahim logs into the system and edits his application.He uploads a clear NID copy and clicks Resubmit.
System Response
The system:
               * updates status to Resubmitted
               * sends the application back to Under Review

At the destination checkpoint, the officer scans Rahim’s QR Code permit.
System Response: The system retrieves permit information and displays:

View Past Visit History (Tourist)
User Action (Tourist):
Rahim logs into the Digital Tourism Permission System and opens his Tourist Dashboard.
From the dashboard menu, he selects Visit History to review the records of his previous trips to restricted tourism destinations.
The system displays a list of all previously approved tourism permits associated with Rahim’s account.
Rahim can view details of each visit including the destination, travel dates, and permit status.
He can also open a specific record to view the full permit information.






System Response: 
                  * Retrieves the visitor's historical permit records from the database
                  * Displays a list of past tourism permits in chronological order
Displayed information includes:
                  * Permit ID
                  * Destination
                  * Arrival Date
                  * Departure Date
                  * Permit Status
                  * Permit Issue Date
Users may also access the previously issued permit document if needed.

Rahim enters:
New Password

Outcome
The implementation of the Digital Tourism Permission System (DTPS) transforms the traditional manual tourism permit process into a secure and fully digital platform.
The system ensures:
                              * faster and more convenient tourism permit applications
                              * secure online document submission and verification
                              * integrated digital payment processing
                              * automated permit generation with QR code verification
                              * real-time application tracking and notifications
                              * centralized tourism data and analytics for authorities

With this system in place, the tourism authority can manage visitor permissions more efficiently while maintaining better control over tourist movement in restricted destinations.
The Digital Tourism Permission System enables a modern, transparent, and efficient tourism management process, improving both the traveler experience and administrative oversight.
